# API层

# etcd交互
通过etcd watch方式和组件通讯

- 所有资源操作在API层变为event，register + wath+ event 方式和组件通讯
- 相较于定义模块接口，event方式具有更好的扩展性，以性能为代价换取其他模块的灵活性

# list-watch
![](./img/2.png)

把所有操作资源的模块看作operator
- operator应该向api-server注册要监听的资源类型
- api-server接收到外部事件后，信息同步到etcd，触发etcd watch 从而触发 operator
- 通过watch方式异步处理而非主动调用
